Dialogs
Introduction
Many actions prompt for user input with dialog. Here are some of the important ones.
  • 
    You can drag the dialog around by grabbing the right border handle or the title area. Some dialogs may move freely, some only move vertically.
  • 
    Many dialogs prompt for element attributes. To reduce the number of keystrokes required, each attribute should be entered as a space separated key value pair on a single line, instead of the standard HTML name="value" format. If the attribute value contains line breaks, use the escaped form: &#xa; instead. For example:
    class x-para
    style width:512px;text-align:center
    tooltips line 1&#xa;line 2
    The same input format is also used for style properties and CSS rules. For example:
    .box {
     width 100%
     border 1px solid #888
    }

The bottom row buttons evaluate the expression and return result in the specified format:
  • =   
    Calculate the result but don't quit.
  • #

  • $

  • OK

  • 

Color Picker
The Color Picker dialog prompt user for a color. The Palette tab () contains a selection of the popular colors. The Names tab list colors by X11 color names () and CSS color names () . Simply click on the tab label to toggle between the two modes. The Picker tab () let you pick color from a HSV gradient field.
Typically, the dialog returns result in a format, hex, dec or name,  as it appears in the input field. The  action change the value to decimal form. The  action change the value to hexadecimal form.
You may enter or select a CSS color name (with the * marker) in the ColorList tab. Note that X11 color that is not a CSS color would stay in numeric form.
You may perform a prefix search for a color name in the ColorList tab by entering a word into the color input field and then press ENTER. The first color with name that starts with the given word would be shown and selected.
Note that hex and name forms are only valid for opaque color. If the color is not opaque, the result is always in decimal, ie. rgba(), form.
  • 
    There is a shortcut for the opacity slider. Click on the opacity label cycle the opacity among 0, 0.25, 0.5, 0.75 and 1.0.

CSS Editor
Use the Files panel Open Document () action to create, browse or edit a CSS file using the CSS file editor. It comes with features like syntax highlighting, content assists, text search, undo/redo, ... etc.
  The content assists use shorthand match, that check for presence of characters in same order as the search string but may have other characters in between, to filter hints. For example, assists for fiz may include font-size and transform-origin-z. That save you from typing “-“ in most cases.
  To work with mobile keyboards, the editor use the Space key to activate the content assist. Press the Space key twice to enter a space.
  The line:column indicator is also an input. Enter location in line or line:column format to jump to the specified location. 1:1 is the first column in the first line.

Image Dimension
The Image Dimension dialog prompt user for output width, height, rotation, ... etc. for the image actions. The output dimension should use the input orientation, ie. before any rotation. If either width/height is missed out, it would be derived from the other dimension and the aspect ratio of the image.
  • Effect
      None, Black and white, Grayscale (
    4, 8, 16 levels
    The first level is always black and the last level is always white.
    ) or 256 levels
  • Adjust (%)  Adjust parameter for effects with value in range 0.0 to 100.0. For Black and white effect, it specify the percentage of black pixels in the output image. Default is 20.0, for
    black on white documents
    For black and white photos, you probably want to use a value around 50.
    .  For Gray (4, 8, 16 levels) effects, it specify the percentage of dark pixels in all but the last level. The last level contains the remaining pixels and is always white. For 256 levels grayscale effect, it specify the percentage of dark pixels to keep, eg. 95.0 specify that the last 5% of light pixels are clipped to white.
  • Quality 0..100. Specify the quality of output for the lossy image formats, eg. JPEG.
  • 
    Note that when turning on grayscale effects, the app. automatically change the output file type to png, which result in lossless and smaller file in most cases.

Photo Dimension
The Photo Dimension dialog prompt user for the output width, height, thumbnail width and height, rotation, ... etc. for the import photo action.
You may optionally create a thumbnail of the imported image by specifying the Tn. width and/or Tn. height inputs.
You may optionally export the photo to the system Photo Album using the  button.
You may also use the Scan Barcode () action to scan for any barcodes in the photo. If found, the barcode messages would be saved to the clipboard.
See the Image Dimension dialog for description of the other fields.
