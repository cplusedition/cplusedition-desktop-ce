    HTML and CSS are mature web technologies for rendering and presenting rich text documents. While they are mostly used for online documents and in view only mode, they work very well in offline and editing mode too. You can create scalable, interactive, rich media, rich text documents, keep them well organized and navigate among them easily using hyperlinks.
    • The app mitigate a lot of security risks and privacy issues by staying completely offline and without any third party content.
    •  The app use WebView API to serve documents directly without going through the network layer.
    . As such it is target at audience who would like to keep the documents offline and who can read and write HTML and CSS.

What permissions are used.
  • Read/write photo and media library - Optional - Required if you want to export or import photos and medias.
  • Camera - Optional. Required if you want to take photos.
  • Microphone - Optional. Required if you want to record audio.
  • Notification - Optional. Required if you want to use reminder alarms.
  • Touch / Face ID - Optional. Required if you want to use biometrics instead of password login.

Does it support scripts.
  • For security and stability reason, the app currently does not support embedding scripts in the documents. Loading of document with the following tags would fail: embed, frame, iframe, object, script.
Does it allow access to local filesystem or external web sites.
  • Access to local filesystem is limited to the assets and documents owned by the app and the photo/media library. No internet access.
    •  Since access is limited to local files, URLs should consist of the file path, query, and fragment parts only. Scheme, host, port, userinfo parts are not valid and may either cause an error or simply discarded.
Why the app fail to take photo, record audio and import photo.
  • The app requires permission to access CAMERA, MICROPHONE, PHOTO/MEDIA library for these actions. Make sure the respective permissions required for the above actions are granted at the Settings/Privacy panel.
Why backspace/delete key not deleting or deleting too much.
  • The backspace/delete key is doing a lot of magic in behind. The backspace/delete key delete not only text but also elements, in particular it may delete some hidden content without showing any visual changes. Be careful when deleting at node boundaries or near some hidden content, like the pop up annotations. If there is any doubt, use the Outline Panel to check out the node structure. In difficult situation, edit the text with the edit node action () instead.
Why restore takes so much longer than backup.
  • Most flash memory system is much slower in writing a large number of small files, as in restore. If you are using incremental backup, restore also need to read and process all the incremental backup files in the sequence. So be patient.
Where is the restored data.
  • The restore data action restore data to the /Restore directory instead of overwriting your existing data at /Home and /Private. You can cherry pick the data that you want to restore by copying files from the /Restore directory to your /Home and /Private directories.
  • Note that since the restored data at /Restore is at a different location, the relative links to files outside the /Restore directory, e.g. ../assets/ in particular, would be broken. However, that would be fixed when you copy the file to its original location at /Home or /Private. That's why you should use absolute path for any reference to /assets in the first place.
What are the differences between the copy/cut/paste actions at the popup editing menu and in the main context menu.
  • The copy/cut/paste action at the popup editing menu are system actions that provide access to the system clipboard. It support cut/paste of text string only. The app also provide a local clipboard that is accessed through the copy/cut/paste actions in the main context menu, and the Outline panel context menu. Content of the local clipboard is not accessible by other apps. It also provide better support for the cut/paste nodes, 
    undo/redo
    actions, option to paste as child or sibling, ... etc. You should use the local clipboard actions whenever possible.
What happen if I deleted /Home/index.html or /Private/index.html.
  • The home action would shows a File Not Found error page. But that is no big deal. You can simply recreate one just like creating any other pages. You can also restore the default home page by copying it from /assets/template/res/index.html.
What are these x-para, x-placeholder, xx-title, fa-home, ... classes.
  • Classes starting with x- are defined in client-v1.css to provide some built-in styles, eg. x-bold, x-para, ... etc. Some of them are marker classes that are used by the application for various purposes. Classes starting with xx- comes from the page template stylesheets. These classes are intended for internal use only, to support the various built-in actions. User should avoid using them directly or creating classes of their own with these prefixes.
  • Classes starting with fa- are standard FontAwesome symbol classes. They are used throughout the application and are included in client-v1.css. User may use these classes for symbols.
    •  Note that you can copy the /assets/css/client-v1.css and css files for the templates to a document directory and export them together with your documents.

Avoid reference to huge image with the IMG element src attribute.
  • If the IMG element reference a huge image with the src attribute, the browser has to load the huge image and then create a scaled version of it for display. That is resource, power and time consuming. If the IMG element is going to display a small size version of a huge image, create a thumbnail of the image for the IMG element instead. Wrap the IMG element in a link to the full size image if necessary. The take photo action works exactly like that. You can create a thumbnail using the ImageViewer export action. 
  • If you has a high density display, consider creating a thumbnail at 2x or 3x the IMG dimension for better image quality, eg. use a 2x thumbnail of 480x360px for an IMG with width:240px;height:180px.
Avoid creating huge documents.
  • Working with huge document require more processing power, resources and power. If possible, split up huge documents and link them together with hyperlinks and/or a table of content page.
How to save photo taken to the system photo gallery.
  • The take photo action now has an Export Photo action to export the full size photo to the system gallery.
How to insert a new paragraph as a sibling instead of as a child of current paragraph.
  • With a collapsed selection, the insert paragraph action () in insert a DIV element at the caret location, which make it a child of the current element. This often frustrates the beginners as you usually want to add a paragraph as a sibling of the current paragraph. To create a new paragraph as a sibling, select the current paragraph element before invoking the insert paragraph action. In difficult situation, do it in the Outline Panel.
How to insert text after an element instead of at the end of an element.
  • Similarly, typing at the trailing boundary of a text node append text inside the text node. To insert text after the boundary, you can use insert text node action (), in the main contextmenu or the outline panel, to insert a text node after the current node.
How to hide the right sidebar.
  • In case you need to interact with your page in the area under the right sidebar, you can toggle the visibility of the right sidebar by fling outward at the right edge or use the  action in the main contextmenu. To hide it permanently, edit the document, select the DIV id=x-rightsidebar element in the Outline Panel and use the hide/show () action in the Outline Panel contextmenu to toggle the visibility of the element. You can even delete it if you want to.
