Dialogs
Introduction
Many actions prompt for user input with dialog. Here are some of the important ones.
  • 
    You can drag the dialog around by grabbing the right border handle or the title area. Some dialogs may move freely, some only move vertically.
  • 
    Many dialogs prompt for element attributes. To reduce the number of keystrokes required, each attribute should be entered as a space separated key value pair on a single line, instead of the standard HTML name="value" format. If the attribute value contains line breaks, use the escaped form: &#xa; instead. For example:
    class x-para
    style width:512px;text-align:center
    tooltips line 1&#xa;line 2
    The same input format is also used for style properties and CSS rules. For example:
    .box {
     width 100%
     border 1px solid #888
    }

Calculator
The Calculator prompt user for a number and allow for simple calculations.
  • C   
    Clear.
  • ~   
    Toggle sign of the last operand.
  • <=  
    Backspace, delete last character.
  • e   
    Exponent, eg. 1e6 for 1000000.
The bottom row buttons evaluate the expression and return result in the specified format:
  • =   
    Calculate the result but don't quit.
  • #   
    Close dialog and return result as integer.
  • $   
    Close dialog and return result with 2 fractional digits.
  • OK  
    Close dialog and return result as is.
  • 
    Note that the calculation result is rounded to no more than 6 decimal digits for readability.
Color Picker
The Color Picker dialog prompt user for a color. The palette tab () contains a selection of the popular colors. The names tab () list colors by X11 color names. The picker tab () let you pick color from a HSV gradient field.
  • 
    There is a shortcut for the opacity slider. Click on the opacity label cycle the opacity among 0, 0.25, 0.5, 0.75 and 1.0.
Date Picker
The Date Picker dialog prompt user for a date, a time or both. You can select how the date/time string is formatted in the format tab (). There is also an option to include the full date/time string as a tooltips attribute.

Photo Dimension
The Photo Dimension dialog prompt user for width, height, thumbnail width and height, rotation, ... etc. for the photo import action.
You may optionally create a thumbnail of the imported image by specifying the Tn. width and/or Tn. height inputs. See the Image Dimension dialog for description of the other fields. If either width/height is missed out, it would be derived from the other dimension and the aspect ratio of the image.
