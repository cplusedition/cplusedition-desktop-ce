The System Toolbar provide actions for backup, restore, print, reset, ... etc.
Login ()
On system that support fingerprint authentication, the fingerprint icon in the login dialog indicates that fingerprint login is available. There must be at least one fingerprint registered in Settings / Security / Fingerprint and screen lock enabled to enable fingerprint login. See Android documentation for details.
When the icon is tinted orange, fingerprint login is available but disabled. When you try to login with fingerprint, you would be prompted to enter the login password to enable fingerprint login. Likewise, when password login is disabled but fingerprint login is enabled, try login with password and you be prompted to enter the fingerprint to enable password login.
When fingerprint login is enabled, click on the fingerprint icons shows the option to disable fingerprint login.
  • 
    Any change to the fingerprint settings, including adding fingers, would invalidate the fingerprint protected login credential. Make sure password login is enabled before you change the fingerprint settings.
  •  Password should consists of ASCII characters between 0x21..0x7e inclusive only.
  •  Note that the login password is app specific. It has nothing to do with the screen lock or device passcode.
  • 
    All system actions except restoring of the master keys require login first. The login password and master keys are created on first login or on restoring of the master keys.
Print
The print action () print the current HTML documents using standard
Android Print Service
, with standard options like Print to PDF, page ranges, ... etc.
  •  Note that the print action load and render the document in its own context according to paper size, orientation, ... etc. The document formatted for print may looks very different from how it appears on screen. So check the preview first.
  •  If you need to print elements that are not visible by default, eg. right side panel, annotations, ... etc, you need to edit the document to force them visible on load before print.
Backup
  • 
    Cancel
  • 
    Perform the backup action, the backup file extension determines what to backup:

    • .ibackup Create an incremental data backup.
    • .backup Create a full data backup.
    • .keys Create a backup of the master keys.
    • .events Create a backup of the calendar events.
    • .zip Create an unencrypted zip archive of a directory..
  • 
    Show context menu that contains the standard file operations (copy, cut, paste, delete, mkdir and rename) as in the normal file picker. See descriptions in the Files Panel for details.
  • 
    For security and stability reason, the unencrypted zip archive cannot be used for restore.
  • 
    The data backup contains only the documents, no keys, and no events. The data backup is protected by one of the master keys, so no password is required.
  • 
    The events backup contains only the calendar events and alarms. The events backup is protected by one of the master keys, so no password is required.
  • 
    The keys backup contains only the master keys. The backup file is protected by a password that should be 8 characters or more.
  • 
    See the Incremental backup section below on how incremental backup works.
You can save the backups to two directories. The Internal backup directory is private and accessible only by this app. It is good for saving snapshots. The External backup directory is located at the primary sdcard, or emulated sdcard. In most case, it should be under / sdcard / Android / data. It can be accessed by any app that has READ/WRITE EXTERNAL STORAGE permission, so you can move the backups to other places, eg. through File managers or adb.  Note that both backup directories would be deleted when you uninstall the app, so you better move your backups to some safer places first.

  • 
    Be sure to create at least one backup of the master keys. The master keys are random generated and unique on each generation. Regeneration create new keys which would not be able to read documents and backups that are protected by the original master keys.
  • 
    Be sure to keep the master keys backup in a private location. The master keys backup is protected by a password. For determined hackers, password protected files are easy to crack if provided unlimited access. Decryption of the masters keys would enable reading of all encrypted data backups and private documents.
  • 
    IMPORTANT: The app deliberately OPT OUT of the standard Android Auto Backup. So none of the documents and backup directories would be backup automatically by Android. You have to manually create backups and move them to some secondary storage regularly.
  •  However, if your system have other backup agents that do not require opt-in and has sufficient privileges, the app cannot prevent them from backing up the app data.
  • Since content of the /External directory is accessible by any app that has READ/WRITE EXTERNAL STORAGE permission, it should be used as a transient storage only. Use the /Internal directory for backups. You can move files between the /Internal and /External directories at any time.
Restore
  • 
    Cancel
  • 
    Perform the restore action. The file extension of the backup file determines what to restore. See the backup action description for details.
    •  Note that keys, events and pdfs are restored/imported as live data. However, data backups are restored to a staging area at /Restore. You can then cherry pick which file or directory to restore by simply copying it to the document directories. The content of the /Restore directory is readonly, but deletable. All data under /Restore are removed when another restore starts.
  • 
    Restore data/events with an another set of master keys provided by a keys backup. The alien keys are used to decrypt the backup file and discarded after the restore action. The restore data is written to storage with the native keys.
  • 
    Restore the master keys. Unlike restoring data, which can be performed any time after login, restoring the master keys can be performed only after a fresh install, or a factory reset. The master key backup file password would becomes the login password after restore.
  • Show file operations context menu. The context menu is the same as above in the backup dialog.

Factory reset
Reset to factory state. This including deleting all private and non-private documents, destroying the login credentials and master keys, ... etc. However, the Internal and External backup directories would stay intact on factory reset.
  • 
    Note that factory reset require login. If you forget the login password or for any reason unable to login, you would have to uninstall the app, which delete everything including the Internal and External backup directories.  Then reinstall the app, copy the backups from secondary storage to the External directory and restore the master keys and data from the backups.
Incremental data backup
The backup/restore data action works in incremental mode if the backup file has an .ibackup extension. Here is how the incremental backup/restore works:
  •  All .ibackup files in a directory are considered as part of an incremental backup sequence. The files should be ordered by filename, in alphabetic order. For example, you can use the date string, like 20160123.ibackup, as filenames.
  •  To create an incremental backup, choose a directory that contains the incremental sequence. Name the backup file with an .ibackup extension and a name that is greater than the last .ibackup file in alphabetic order. The backup action backup only files that are modified since the previous .ibackup file, if there is one. Otherwise, it backup all files as if it is a full backup.
  •  To restore from an incremental sequence, select an .ibackup file as the source file. The restore action restore all the .ibackup files in the directory before and including the selected file in filename order.
  • 
    The incremental backup use file size and timestamp to check for modified files.
  • 
    Note that you can restore from any .ibackup instead of just the last one. Each sequence up to a particular .ibackup file represent a snapshot at the time that .ibackup file was created.
No backup
You can exclude a directory and everything under it from being backed up by creating a marker file in the directory. To create/delete such marker file, use the
Toggle Nobackup
action () in the Explore Files panel contextmenu.
System info
Audio recording status ()
When an audio recording is in progress, a microphone icon would be shown in the Main Toolbar. Click on the microphone icon stops the recording.
Audio playback status ()
When an audio playing is in progress in background, a speaker icon would be shown in the Main Toolbar. Click on the icon shows the
Audio Player
where you can control the playback again.
