Events Panel
Introduction
The Events Panel show the calendar events. If an event has the reminder alarm enabled, it sends a notification at the scheduled time of the event. Each event entry has following fields:
  •  The alarm/done indicator:
      means event is not done and has a pending alarm.
      means event is not done and has a pending alarm. However, it is not yet posted to the notification system for some reasons.
      means event is not done and has no pending alarm.
      means event is marked as done. When an event is marked as done, its alarm would be disabled and, if it is a repeating alarm, its time stamp would stop advancing.
    •  Click on the indicator toggle the done status.
  •  The date/time of the event.
  •  The repeat setting (once, daily, workdays, weekly, monthly or yearly).
  •  The private indicator () means event is private and would only be shown when user is logged in.
  •  Finally the description text for the event. If the event has an associated document location, this field is shown as a link and click on it goes to the target location. The link is colored red if the target document does not exists.
  •  You can create events/alarms here. Or you can use the insert event action in the main context menu which automatically attach the document location to the event.
  •  Note that when the alarm goes off, it would post a notification to remind you of the event. But it would not wake up the device or wake you up like an alarm clock.
  •  Note that the system post a restriction on the number of pending alarms. As of iOS 11, the limit is 64. But that is not a problem. The app make sure alarms with the earliest triggers are posted first.
  •  Unfortunately, the iOS notification system do not has the necessary support to automatically update every kind of repeating alarms the app supported. So repeating alarms, after the first trigger, may require the app running in the foreground to schedule the next trigger. Has the app running any time before the next trigger would get it properly rescheduled.
