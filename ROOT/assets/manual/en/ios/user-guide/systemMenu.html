The System Toolbar provide actions for backup, restore, reset, ... etc.
Login
On system that support Touch / Face ID, the fingerprint icon in the password login dialog indicates that biometric login is available. Enter the login password and click on the fingerprint icon to enable biometric login.
When biometric login is enabled, the login action always prompt for the biometric first. If biometric login failed or is cancelled, it fallback to password login. 
When biometric login is enabled, the password login dialog shows the option (❌) to disable biometric login.
  • 
    Any change to the Touch / Face ID settings, including adding fingers, may invalidate the biometric protected login credential. Make sure password login is enabled before you change the biometric settings.
  •  Password should consists of ASCII characters in range 0x21..0x7e inclusive only.
  •  Note that the login password is app specific. It has nothing to do with the screen lock or device passcode.
  •  For security reason, all login credentials and keys stays at the native side and protected by device only keys. As such all biometric and password dialogs are implemented at the native side using native UI.
Print
  • 
    Print current HTML document through system Print Service.
  •  Note that you may save the print output as PDF in the print preview page. First, use zoom guesture on the print preview page to open the preview as PDF. Then use the Share action in the PDF viewer to save the PDF to a file or share it with other apps.
  •  Note that printing render the document in its own context according to the print options, like paper size, orientation, ... etc. It may look very different from what it appears on screen. Some layout that works fine on screen may not work well on the print media. So check the preview first and you may have to tweak the document for a proper printout.
  •  Printing render the document as it is loaded. To print elements that are not visible by default, like pop up annotations, you may have to tweak the document to force them visible on load before print.

Backup / Restore
Backup keys
Context menu
  • 
    Cancel
  • 
    Perform the backup action, the backup file extension determines what to backup:

    • .ibackup Create an incremental data backup.
    • .backup Create a full data backup.
    • .keys Create a backup of the master keys.
    • .events Create a backup of the calendar events.
    • .zip Create an unencrypted zip archive of a directory. Note that files, including encrypted files in the /Private directory, are exported as unencrypted.
  • Show context menu that contains the standard file operations (copy, cut, paste, delete, mkdir and rename) as in the normal file picker. See descriptions in the Files Panel for details.
  • 
    For security and stability reason, the unencrypted zip archive cannot be used for restore.
  • 
    The data backup contains only the documents, no keys, and no events. The data backup is protected by one of the master keys, so no password is required.
  • 
    The events backup contains only the calendar events and alarms. The events backup is protected by one of the master keys, so no password is required.
  • 
    The keys backup contains only the master keys. The backup file is protected by a password that should be 8 characters or more.
You can save the backups to two directories:
  •  The Internal backup directory is app private and accessible only by this app. It is good for saving snapshots.
  • 
    The External backup directory is the gateway to the outside world. It is accessible through iTunes File Sharing on your computer and the
    Files app
    on the device. You can easily move files in the External directory to/from your computer or a cloud drive. On the other side, the Backup/Restore action and the Import/Export actions in the Files panel can move backups and media files (video, audio, image, PDF) in the External directory to/from your document directories on the device.
    Note that both backup directories would be deleted when you uninstall the app. So you better move your backups to some safer places first.
  • 
    Note that the app deliberately opt out of iTunes backup from backing up any app data. So you have to create backups and move them to secondary storage manually.
  • 
    Be sure to create at least one backup of the master keys. The master keys are random generated and unique on each generation. Regeneration create new keys which would not be able to read documents and backups that are protected by the original master keys.
  • 
    Be sure to keep the master keys backup in a private location. The master keys backup is protected by a password. It is vulnerable to attack if provided unlimited access, unless you use a really really good password 🤔. Decryption of the masters keys would enable reading of all encrypted data backups and private documents.
Restore
  • 
    Cancel
  • 
    Perform the restore action. The file extension of the backup file determines what to restore. See the backup action description for details.
  • 
    Restore with alien key action. SImilar to the normal restore action, but this action prompt for a keys backup file. It use the provided keys to decrypt the data backup and re-encrypt the data using its own master keys, if necessary, before writing it to the /Restore directory.
  • 
    Restore the master keys. Unlike restoring data, which can be performed any time after login, restoring the master keys can be performed only after a fresh install, or a factory reset. The master key backup file password would becomes the login password after restore.
  • Show file operations context menu. The context menu is the same as above in the backup dialog.
  •  Note that keys and events are restored as live data. However, documents are restored to a staging area at /Restore. You can then cherry pick which file or directory to restore by simply copying it to the document directories. The content of the /Restore directory is readonly, but deletable.
Change password
Change password
Login password must be 4 characters or more. You may change the login password anytime. It does not affect the master keys. When the master keys are destroyed on factory reset, the login password is also destroyed.
  • 
    Disable password login. This is only available when both password and biometric login are enabled.
Reset user settings
Reset user settings
Factory reset
Reset the user settings to default values. All documents, events, login password and master keys would stay intact.
Factory reset
Reset to factory state. This including deleting all private and non-private documents, destroying the login credentials and master keys, ... etc. However, the Internal and External backup directories would stay intact on factory reset.
  • 
    All system actions except restoring of the master keys require login first. The login password is created on first login or on restoring of the master keys.
  • 
    If you forget the login password, you would have to perform a factory reset and restore the master keys and data from the backups.
Incremental data backup
The backup/restore data action works in incremental mode if the backup file has an .ibackup extension. Here is how the incremental backup/restore works:
  •  All .ibackup files in a directory are considered as part of an incremental backup sequence. The files should be ordered by filename, in alphabetic order. For example, you can use the date string, like 20160123.ibackup, as filenames.
  •  To create an incremental backup, choose a directory that contains the incremental sequence. Name the backup file with an .ibackup extension and a name that is greater than the last .ibackup file in alphabetic order. The backup action backup only files that are modified since the previous .ibackup file, if there is one. Otherwise, it backup all files as if it is a full backup.
  •  To restore from an incremental sequence, select an .ibackup file as the source file. The restore action restore all the .ibackup files in the directory before and including the selected file in filename order.
  • 
    The incremental backup use file size and timestamp to check for modified files.
  •  Note that you can restore from any of the .ibackup files in an incremental sequence, not just the last .ibackup file. It restore a snapshot up to the time that .ibackup file is created.
No backup
You can exclude a directory and everything under it from being backed up by creating a marker file in the directory. To create/delete such marker file, use the
Toggle Nobackup
action () in the Explore Files panel contextmenu.
