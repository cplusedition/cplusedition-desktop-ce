2.12.1
for iOS
v 2.12.1
This release fixed a few minor issues. Furthermore, the Photo (Wall) template now has a button to cycle through a few thumbnail sizes.
  • 
    To work with the wide variety of screen sizes, the Photo (Wall) template now has a thumbnail size button () that allow you to choose the proper thumbnail size for your device. 
    •  Unfortunately, this seems to hit a bug in the CSS engine when working with SVG images. See Known Issues section below for details.
    •  ImagePicker now support SVG images.
    • Fixed a few bugs and some UI issues.
      Known Issues
      • When the Photo (Wall) page thumbnail size is changed, all images including SVG,  are resized properly. However, the container for a SVG image does not. Fortunately, this is only a transient problem. When the page is refreshed, eg. on save or some major layout changes, the problem is fixed and the page is rendered properly.  This is probably an upstream WebView issue.
        v 2.12
        This release automated the tedious task of creating an index/gallery page for selected files under a directory. The app now stays inside the safe area of the latest iPhones. There is also a new Photo (Wall) template, and a few other improvements. Furthermore, it worked around a problem that may cause a page to offset incorrectly in landscape orientation and a problem that may cause the main toolbar not responding.
        • 
          There is now a Generate Index/Gallery Page () action in the Files panel context menu that create an index page for selected files under a directory, optionally recursively. Using an Audio template, it create a page with all the selected audio and video files. Using a Home template, it create an index page with links to all selected HTML and media files. Using a Photo (Sticker1) or Photo (Wall) template, it create a gallery page with all the selected image files.
        • There is now a Clone File () action in the Files panel context menu that works like a Paste File action, but preserving the file timestamp.
            Note that incremental backup use file size and file timestamp to determine whether a file is updated. Copying files with an old timestamp may prevent it from being included in an incremental backup. So use with care.
        • The Share () action in the Files panel context menu is replaced with the View () action that open the media file using QuickLook, where you can share it if you want to. 
        • The Files panel now stays at the last shown directory, instead of always showing the directory containing the current document, on open. Moreover, the Files panel toolbar now has the current directory () button that allow you to  go to the directory for the current document quickly.
        • The Back () and Forward () action in the main toolbar now transverse over HTML files only, skipping the media files, which often take a long time to load and may require an external viewer.
        • 
          The Insert Link () dialog now automatically fill in the text field using the file name you enter in the URL field if the text field it is empty.
        • The video player is now paused instead of destroyed when the app going into background. It can resume playback immediately without having to reload on resume. Moreover, the app now accept .m4v video files.
        • 
          The Recents symbol page now sort symbols in a stable order instead of on access time so looking up symbol is easier.
        • 
          There is a new Photo (Wall) template that should work better than Photo (Sticker1) for larger screen and it is now the default. You can use the other variants through the Templates panel.
        • The Photo (Sticker1) template now hides the stacking () and move () buttons in browsing mode.
        • 
          The Audio template now has a status symbol before the description which can be used to mark if the media has been viewed, or for any other purpose you designated.
        • 
          The Audio Info and Video Info blocks created through the Main context menu now use a transparent background and without border so it can easily blend into various places.
            This affect appearance of existing media info blocks created through the Main context menu.
        • 
          The Calculator is now smarter and use the initial value as an operand if the first input is an operator. If the first input is a value key, then it ignore the initial value and use the new input value.
        • 
          The app now stays inside the safe area of the latest iPhones.
        • Worked around a problem that may cause a page to offset incorrectly in landscape orientation and a problem that may cause the main toolbar not responding. Fixed a few other minor issues.

            v 2.11.0
              • This release contains many time saving improvements and a few critical bug fixes.

              • 
                PromptSymbol dialog now has a Recents panel that keep the most recently used symbols.
              • 
                There are actions in the Files panel context menu to record audio (), record video () or take photos () directly to a file.
              • 
                There is also a new Generate QR Code () action in the Files panel context menu that generate a QR code image for some text payload you specified.
              • 
                The Scan Barcode () action now recognize all kinds of barcode that is supported by the Vision Framework instead of just QR code.
              • 
                You may now apply grayscale effect with 8, 16 and 256 levels, in addition to the 2 and 4, when saving images.
              • 
                Insert image dialog URL input now has an Import From Photo Library () button that import and insert an image directly from the Photo Library.
              • You may now use long press to extend the selection in the Outline panel instead having to click the Extend Selection () button repeatedly.
              • 
                The FilePicker gets a Goto Current Directory () and Goto Target Directory () actions that goes to the directory that contains the document being edited and the directory that contains the initial link target respectively.
              • 
                The DatePicker also gets a Goto Today () and Goto Target Date () actions. The target date is the initial date when opening the dialog.
              • 
                Opening a PDF file now opens it in QuickLook directly instead of having to go through the DocumentInteractionController.
              • 
                The default sidebar tab now links to index.html instead of /Home/index.html which is often more useful.
              • 
                You can now set a custom background color for each individual event in the Events panel.
              • 
                There is a new home-simpler template, which is a simplified version of the original home template, and a photo-sticker1 template that use stickers for photos and images.
              • 
                The blog template get a few new backgrounds.
              • 
                The Move Sticker () action now works in browsing mode too.
              • 
                The Rename File () action now allow changing the file ext between all lower case and all upper case.
              • 
                Delete File () actions now create backups at the /Recover directory.
              • 
                The Record Video and Record Audio actions now create a media info block even on cancel using a sample media as placeholder. You can edit it to link to your own media.
              • 
                The canvas history dialog now has an Edit Text () action to edit a Text node in the drawing.
              • 
                There a few other minor improvements.
              • 
                Fixed crashes that may occur when processing some invalid image files, a problem that may cause the main toolbar not responding and a few other minor bugs. Moreover, the problem with dragging text selection handle is fixed as of iOS 12.2.
