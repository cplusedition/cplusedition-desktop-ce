v 2.14.2
This release added a few shortcut gestures. In particular, you may now open and edit a CSS file associated with a document with a single fling, putting CSS file editing front and center in the workflow. 
  • 
    Fling inward from the right edge opens/edits a CSS file. At the top half, it always open a CSS file picker. At the bottom half, it look for the last editable style sheet linked to the document and edits it if one is found. Otherwise, it opens a CSS file picker. If you select a new CSS file that is not already linked to the document and the document is in editing mode, a link the CSS file is automatically added to the document. To manage the links to css files, use the
    Links panel
    .
  •  Fling inward from the left edge under the tabs now opens the Files panel if document is in browsing mode. If document is in editing mode, it opens the Outline panel.
  •  Improved the fling gesture detection to work simultaneously with drag scrolling. It also require you to fling faster to avoid triggering when working with text selection.
  •  Disabled auto correct and auto complete at most dialog inputs.

                    v 2.13
                    This release added a CSS file editor with syntax highlighting, content assists, text search, undo/redo .. etc.
                    • 
                      For those who would like to do some heavy lifting with CSS, there is now a CSS file editor that comes with features like syntax highlighting, content assists, text search, undo/redo, ... etc. The value hints even includes the color and font names and a few shortcuts that save you some keystrokes when working with mobile keyboards. There are also context actions to insert color, font or file path through a picker. See the Dialogs page in the User Guide for more details.
                      This replace the stylesheet editor that works directly on the parsed CSS rulesets in the document. The CSS file editor works on the CSS file instead. As a consequence, changes would not be applied to the document that use the CSS file until the document is reloaded. 
                      You may now create, view or edit CSS files from the Files panel by simply specifying a file name with .css extension.
                        The content assists use shorthand match, that check for presence of characters in same order as in the search string but may have other characters in between, to filter hints. For example, assists for fiz may include font-size and transform-origin-z.
                        To work with mobile keyboards, the editor use the Space key to activate the content assist. Press the Space key twice to enter a space.
                    • Fixed a few minor issues.
